# 企微文档表格粘贴问题修复总结

## 问题描述

从企业微信文档粘贴表格内容到 Markdown 编辑器时，存在以下问题：

### 已修复问题 ✅

1. **合并单元格表格图片位置错误**
   - 表格中的图片被显示到第一行，而不是正确的合并单元格中
   - 原因：`cleanHTML()` 函数移除了 `colspan` 和 `rowspan` 属性
   - 解决：保留这些关键属性，只移除宽度相关的样式

2. **项目符号不被识别**
   - 表格内的列表项（如 `● 项目`）没有被正确换行
   - 原因：只检测了 HTML `<ul>/<ol>` 标签，没有检测文本标记
   - 解决：添加 `hasTextListMarkers()` 和 `processTextListWithMarkers()` 函数

3. **Office 标签显示**
   - 图片下方显示 `<o:p></o:p>` 标签
   - 解决：在 `cleanHTML()` 中添加移除逻辑

4. **扩展列表标记支持**
   - 支持更多企微常用符号：● ○ ▪ ■ • ◆ ◇ ▲ ▼ ▶ ◀ ★ ☆ ✔ ✘ → ➤
   - 支持数字标记：1、 2、 1. 2. 1) 2)
   - 支持字母标记：a) b) A) B)
   - 支持圆圈数字：① ② ③ 等

5. **表格自适应样式**
   - 文本列过长导致图片列被压缩
   - 解决：在 `markdown.js` 中添加列分析，为文本列设置 `max-width` 和强制换行

### 已知问题 ⚠️

1. **`<br>` 标签在 Typora 中显示**
   - 表格内换行的地方保存后用 Typora 打开会显示 `<br>` 文本
   - 原因：Markdown 表格和 Typora 对换行的处理方式不同
   - 状态：暂不修复

2. **`-` 标记显示为 `\-`**
   - 使用 `-` 作为项目符号时，会被 Markdown 解析器转义
   - 解决方案：代码中使用 `•` (U+2022) 代替 `-`
   - 状态：已部分解决（新生成内容使用 `•`）

## 技术实现

### 核心文件修改

#### `src/utils/turndown.js`

**新增函数：**

1. `hasTextListMarkers(text)` - 检测文本是否包含列表标记
2. `processTextListWithMarkers(text)` - 处理文本列表标记，转换为换行
3. `processCellContent(cell)` - 处理表格单元格内容（支持列表、图片）
4. `processListInTable(listNode)` - 处理表格内的 HTML 列表
5. `processParagraphWithLists(pNode)` - 处理包含列表的段落
6. `processImageInCell(imgNode)` - 处理单元格内的图片

**修改函数：**

1. `cleanHTML(html)` - 清理 HTML 时保留表格关键属性
2. 添加自定义表格处理规则 - 支持 colspan/rowspan 和列表

#### `src/utils/markdown.js`

**修改内容：**

// 保留 colspan 和 rowspan
const colspanMatch = attrs.match(/\scolspan\s*=\s*["']?(\d+)["']?/i)
if (colspanMatch) {
  keptAttrs += ` colspan="${colspanMatch[1]}"`

### 列表标记检测

```javascript
function hasTextListMarkers(text) {
  // 符号标记
  const symbols = ['●', '○', '▪', '■', '•', '◆', '◇', '▲', '▼', '▶', '◀', '★', '☆', '✔', '✘', '→', '➤']
  for (const sym of symbols) {
    if (text.includes(sym)) return true
  }
  // 数字标记：1、 1. 1)
  if (/\d+[、.)]\s/.test(text)) return true
  // 字母标记：a) A)
  if (/[a-zA-Z][、.)]\s/.test(text)) return true
  return false
}
```

### 表格列分析

```javascript
// 分析每一列是否包含图片
const columnHasImage = new Array(headerCells.length).fill(false)

function analyzeRow(cells) {
  for (let j = 0; j < cells.length; j++) {
    if (cells[j].includes('![](') || cells[j].includes('<img')) {
      columnHasImage[j] = true
    }
  }
}

// 为不同类型的列设置不同样式
if (columnHasImage[j]) {
  cellStyle = 'min-width:120px;max-width:350px;'
} else {
  cellStyle = 'max-width:350px;min-width:80px;word-wrap:break-word;overflow-wrap:break-word;white-space:normal;'
}
```

## 经验总结

1. **HTML 转换时需谨慎清理属性**
   - 表格的 colspan/rowspan 对结构至关重要，不能随意删除
   - 应该只移除影响自适应的属性（如 width）

2. **文本列表标记检测需要全面**
   - 不同平台的列表标记格式多样
   - 需要同时检测符号、数字、字母等多种格式

3. **Markdown 表格的局限性**
   - 标准 Markdown 表格不支持多行单元格
   - 需要使用 HTML `<br/>` 实现换行
   - 不同工具对 `<br/>` 的支持程度不同

4. **使用 Unicode 符号避免转义**
   - `•` (U+2022) 不会被 Markdown 解析器处理
   - 避免使用 `-` 等可能被转义的字符

5. **列宽度控制需要分类处理**
   - 图片列需要最小宽度保证显示
   - 文本列需要最大宽度强制换行
